# 3.1 El servicio de directorio

## Documentación LDAP

### ¿Qué es LDAP?
<p>LDAP son las iniciales de Lightweight Directory Access Protocol (protocolo ligero de acceso a directorios). Como su nombre indica, es un protocolo pensado para permitir el acceso a servicios de directorio.</p>
<p>LDAP funciona con un esquema cliente-servidor, en el que uno (o varios) servidores mantienen la misma información de directorio (actualizada mediante réplicas) y los clientes hacen consultas a cualquiera de ellos.</p>
<p>En LDAP la información está estructurada en forma de árbol. Esta estructura se llama directorio y cada nodo se llama entrada . Por su parte, cada entrada está formada por un conjunto de atributos, cada uno de ellos es de un tipo y contiene uno o más valores.</p>

### Características de LDAP
<ul>
<li>Consulta rápida de datos : el objetivo de un servidor de directorio es la recuperación rápida de información. LDAP ofrece un método ligero de conexión a un repositorio, lectura de datos y cierre de la conexión.</li>
<li>Solución distribuida : puesto que LDAP es un protocolo nativo de red de arquitectura distribuida, puede distribuir a toda la red información lista para ser usada por todas las aplicaciones. Puede incluso reproducir una parte del directorio para dar un servicio con ciertas características en un entorno determinado. O mover la información a varios sitios sin afectar ningún acceso externo a estos datos.</li>
<li>Independencia de la plataforma: como LDAP es un protocolo estándar, que proporciona un método de acceso a datos remoto y local, es posible intercambiar completamente la implementación de LDAP sin afectar a la forma en que se podrá acceder a los datos.</li>
<li>Seguridad integrada en el repositorio: la información de acceso se almacena en el mismo repositorio. Algunos productos pueden trabajar con derechos y permisos de fina granularidad y también proporcionan la capacidad de controlar los derechos de acceso en función de factores externos. Dado que estos controles se ejecutan de forma centralizada por el servidor, son fáciles de mantener.</li>
<li>Implementación fácil del cliente: la disponibilidad de interfaces de programación ( API ) de LDAP para casi cualquier lenguaje de programación facilita la compatibilidad de LDAP con prácticamente todas las aplicaciones.</li>
<li>Gran difusión: como LDAP es un protocolo estándar, muchos proveedores lo han adoptado en su software. Por ejemplo, IBM Tivoli, Microsoft Active Directory, Novell eDirectory o RedHat Directory Server.</li>
<li>Bajo coste : LDAP es un protocolo estándar y no es necesario pagar licencias por su uso ni para disponer de clientes o servidores (de código fuente abierto).</li>
</ul>
